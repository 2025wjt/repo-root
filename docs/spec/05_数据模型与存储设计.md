# 数据模型与存储设计

## 1. 总体存储原则

### 1.1 控制信息与业务产物分离

#### 控制信息进入数据库
包括：
- 项目状态
- 任务状态
- 审批状态
- Agent 状态
- 事件记录
- 运行日志索引

#### 业务产物进入文件系统
包括：
- 结构化需求单
- PRD
- 技术方案
- API 说明
- 任务清单
- 测试报告
- 交付说明

### 1.2 数据库存“事实”，文件系统存“内容”
- 数据库记录状态、关系、索引、引用路径
- 文件系统保存文档正文和导出包

## 2. 核心对象模型

比赛版最少冻结以下对象：
- Project
- Task
- Approval
- Event
- Artifact
- AgentRuntimeStatus

## 11. 命名规则

### 文档命名
`<artifact_name>_v<version>.<ext>`

示例：
- `prd_v1.md`
- `architecture_design_v1.md`
- `task_list_v1.json`
- `delivery_note_v1.md`

### 模块型产物命名
`<module_name>_<artifact_type>_v<version>.<ext>`

示例：
- `login_impl_v1.md`
- `todo_test_report_v1.md`

## 12. 版本规则

- Project：整体版本
- Artifact：每次修改递增版本
- Task：返工或重试建议新建任务或新版本
- Approval：重新审批时新建记录，不覆盖旧记录
