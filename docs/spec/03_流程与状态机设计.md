# 流程与状态机设计

## 1. 流程设计目标

- 保证流程闭环
- 保证治理可控
- 保证人工决策可嵌入
- 保证失败可回流
- 保证前端可展示

## 2. 主流程阶段

比赛版主流程定义为 6 个阶段：

1. 项目创建阶段
2. 需求对齐阶段
3. 方案确认阶段
4. 开发测试阶段
5. 结果评审阶段
6. 交付完成阶段

## 3. 节点定义

### 节点 1：需求提交
- 输入：用户原始需求
- 输出：项目实例、原始需求记录

### 节点 2：需求澄清
- 执行者：Clarify Agent
- 输出：结构化需求单、待确认项

### 节点 3：PRD 生成
- 执行者：PM Agent
- 输出：PRD、验收标准、优先级说明

### 节点 4：PRD 审批
- 执行者：Human Reviewer
- 结果：approved / rejected
- rejected 时返回 PRD 修改节点

### 节点 5：技术方案生成
- 执行者：Architect Agent
- 输出：技术方案、模块拆分、接口说明、任务清单

### 节点 6：技术方案审批
- 执行者：Human Reviewer
- 结果：approved / rejected
- rejected 时返回技术方案修改节点

### 节点 7：开发任务执行
- 执行者：Dev Agent
- 输出：模块实现结果

### 节点 8：测试验证与返修
- 执行者：Test Agent / Dev Agent
- 输出：测试报告、缺陷说明、返修结果

### 节点 9：最终评审与交付
- 执行者：Human Reviewer / Orchestrator
- 输出：最终评审结果、交付说明、项目完成状态

## 4. 主流程图

```text
需求提交
  ↓
需求澄清
  ↓
PRD 生成
  ↓
PRD 审批
  ├─ 驳回 → 返回 PRD 生成
  └─ 通过 → 技术方案生成
              ↓
          技术方案审批
          ├─ 驳回 → 返回 技术方案生成
          └─ 通过 → 开发任务执行
                      ↓
                  测试验证
                  ├─ 失败 → 返修 → 重新测试
                  └─ 通过 → 最终评审
                              ├─ 驳回 → 返回指定返修节点
                              └─ 通过 → 交付完成
```

## 5. 项目状态机

建议项目状态：

- `created`
- `clarifying`
- `prd_generating`
- `waiting_prd_approval`
- `designing`
- `waiting_design_approval`
- `developing`
- `testing`
- `reviewing`
- `delivering`
- `done`
- `failed`
- `paused`

### 项目状态流转

```text
created
  → clarifying
  → prd_generating
  → waiting_prd_approval
    ├─ rejected → prd_generating
    └─ approved → designing
                  → waiting_design_approval
                    ├─ rejected → designing
                    └─ approved → developing
                                  → testing
                                    ├─ repair_loop → developing
                                    └─ all_passed → reviewing
                                                  ├─ rejected → developing/testing
                                                  └─ approved → delivering → done
```

## 6. 任务状态机

建议任务状态：
- `pending`
- `dispatched`
- `processing`
- `waiting_approval`
- `blocked`
- `waiting_repair`
- `retrying`
- `done`
- `failed`
- `cancelled`

## 7. 审批状态机

建议审批状态：
- `pending`
- `approved`
- `rejected`
- `expired`

## 8. Agent 状态

建议 Agent 状态：
- `idle`
- `busy`
- `waiting_input`
- `degraded`
- `offline`

## 9. 状态写权限

### Orchestrator 可修改
- 项目状态
- 任务状态
- 审批记录创建
- 回流任务创建
- 事件记录写入

### Agent 只允许修改
- 自己的运行中状态
- 任务结果输出
- 产物文件
- 执行摘要

### 前端只允许触发
- 创建项目
- 提交审批动作
- 查询项目/任务/产物
- 可选暂停/恢复

## 10. 失败回流规则

### 文档类回流
- PRD 审批驳回 → 创建 `revise_prd`
- 技术方案驳回 → 创建 `revise_architecture`

### 开发类回流
- 测试失败 → 创建 `repair_module`
- 修复后再创建 `retest_module`

### 回流原则
- 局部回流优先
- 单模块失败不影响已通过模块
- 单模块返修次数建议上限 2 次

## 11. 强门禁

比赛版至少保留 3 个门禁：
- PRD 审批门
- 技术方案审批门
- 最终评审门
